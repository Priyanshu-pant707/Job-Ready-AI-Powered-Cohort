# Bundle Optimization :


-  it means reducing the size , number , and loading cost of Javascript/css bundles sent to the browser


## Goal :
- faster page load.
- Better core web vitals
- Less JS execution
- happier users




## Why bundle optimizatio is critical ?
- large JS bundle causes :
 - slow time to first paint (FCP)
 - slow time to interactive (TTI)
 - blocking main thread
 - poor mobile performance

# How to reduce bundle size ?
 - code splitting,tree shaking , lazy loading ,minification.

##  why is js more expensive than images ?
 - js must be downloaded ,parsed,compiled,and executed
### What is vender chunking ?
  -  separating third-party libraries into cacheable bundles




  - Summary :
     - bundle optimization focuses on reducing javascript and asset payload sent to the browser. Technique include code splitting , lazy loading, tree shaking , and minification. The goal is to improve load time , reduce main-thread work, and enhance Core Web vitals.
